# **【检测】【XX客户】lora_a 精度异常问题定位案例**

## 一、问题背景

lora_a算子单核运行精度异常，怀疑内存问题，计划采用内存检测工具检查算子有无相关内存问题



## 二、排查思路

1. 检查算子是否存在内存越界读写问题
2. 检查算子是否存在核内竞争问题



## 三、定位过程

 mssanitizer -t racecheck --log-file=error-log-fa-del-process python3 test.py

![image-20251212100305749](../pic/517b93326b853fab3f119e7b8ee6e6f1_1075x77.png)

分析过程：

1. 在aiv 0核UB上存在PIPE_V和PIPE_MTE2的竞争问题，代码运行逻辑如下：
   PIPE_V先对UB上地址0x18ea0写入，之后PIPE_MTE2对UB地址0x18620写入；对应工具日志如下：

![image-20251212100331998](../pic/84f451b0430420ea3c923f9bb8f5bdfd_1078x28.png)

![image-20251212100342397](../pic/81051157b6eba9f14d80e1e938218d31_1076x24.png)

对应代码段位置：PIPE_MTE2对应代码在前，PIPE_V在后，两者代码外部有2层循环；说明当前循环的PIPE_V和下一次循环的PIPE_MTE2之间忘记插入同步出现竞争问题；
PIPE_MTE2写入UB范围：[0x18620, 0x18620 + 512 * 32]
PIPE_V写入UB范围：[0x18ea0, 0x18ea0+4]
两者存在地址交叉，又由于忘记加同步，导致竞争问题

算子开发认为两次循环间的PIPE_V和PIPE_MTE2写入的UB地址不应有重叠，推断出精度问题的原因是PIPE_MTE2搬运参数设置不合理，修复该问题之后精度正常

## 四、总结

1. 存在竞争问题包含两重含义：
   （1）：两条pipe间未添加同步指令
   （2）：pipe（联调）对某个空间的地址存在交叉操作，并且至少有一条pipe对该地址进行了写入
