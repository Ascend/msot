# **【检测】【XX客户】flash_attention_train 算子UB越界写导致算子挂掉问题定位案例**

## 一、问题背景

flash_attention_train算子在q_head_num=8时，算子运行异常，报ai_core_error。
q_head_num=1时，算子运行正确且精度符合要求



## 二、排查思路

怀疑内存问题，这会导致程序异常中断
导致程序异常的内存问题如下：

1. 对齐问题，对齐问题主要在UB/L0A/L0B/L0C/L1等片上内存

2. 地址越界问题，目前910B系列芯片UB（统一缓冲区）空间为192KB，其他片上内存空间的详细信息请参考CANN包中的以下文件

/usr/local/Ascend002/ascend-toolkit/8.0.RC2.alpha002/aarch64-linux/data/platform_config
